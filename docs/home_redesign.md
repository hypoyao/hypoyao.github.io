# 首页设计稿（v1）——从“小游戏门户”到“创作展示平台”

参考图：你上传的截图（左侧 Hero 输入框 + 右侧优秀作品展 + 卡片内折叠 Prompt 面板）。

## 目标
1. 第一眼让用户感知：这里是**用 AI 创造游戏**的平台，而不是“游戏聚合站”。
2. 强化转化路径：从“看”到“试”只需 **1 步**（首屏输入一句话 → 进入创作）。
3. 强化“创作痕迹”：每个作品卡片展示可折叠的 **创作思考（Prompt）**。

## 信息架构（IA）
**Top Bar**
- 品牌：AI创意小游戏 + 副标题「把“对话”变成“作品”」
- 右侧动作：创作我的游戏（主按钮）+ 头像/登录入口

**Hero（首屏核心）**
- 主标题：用 AI 对话，把奇思妙想变成你的专属小游戏
- 副文案：让孩子看到“思考的过程”。让家长看到“创造的证据”。
- 主要 CTA：输入框（prompt）+「开始创造」按钮（GET 跳转到 /create?prompt=...）
- 快捷模板：3 个 chip（可扩展更多），降低空白输入门槛

**优秀作品展（右侧/次级首屏）**
- 标题 + 副标题
- 2×2 卡片（可按最新/推荐/编辑精选）
- 卡片内容：
  - 封面（4:3）
  - 标题
  - 创作者（头像 + 名称）
  - 折叠面板：创作思考（Prompt）摘要

**全部作品**
- 标题 + 副标题
- 3 列网格（移动端降为 2 列）
- 卡片内容：
  - 封面（1:1）
  - 标题、短描述
  - 创作者（头像 + 名称）

## 视觉语言（与参考图一致）
- 结构：**左 Hero / 右 Showcase** 的双栏首屏（移动端自动堆叠）
- Hero 背景：蓝/绿的柔和渐变光晕 + 白色玻璃拟态卡片
- 强 CTA：彩色渐变按钮（天蓝 → 绿色）
- Prompt 面板：折叠 `details/summary`，低干扰但可随时展开查看“过程”

## 数据策略（落地）
- Prompt 等游戏文件统一从数据库读取（`game_files` / `creator_draft_files`），不再依赖 `public/games/*`
- 没有 prompt 时显示占位文案：暂未公开 Prompt（后续可自动保存创作对话记录）
- 首页输入框会把 prompt 传到 `/create?prompt=...` 并预填充在创作页输入框中
